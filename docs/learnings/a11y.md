# Accessibility

## aria-live

- **Never put `aria-live` + `aria-atomic="true"` on a container that re-renders per-item.** During `starAll`, this announces every repo change. Place `aria-live="polite"` on the leaf `<p>` showing the current action instead.
- **For step-transition announcements, use a static string** (not dynamic counts) so `aria-live="assertive"` fires once per step change, not once per tick.
- **`aria-label` on a bare `<div>` is ignored.** Without a semantic role, screen readers skip the label. Add `role="status"` (implicitly `aria-live="polite"`) or another appropriate role.

## Focus management

- **`aria-hidden="true"` alone doesn't prevent keyboard focus** on descendant `<a>`/`<button>` elements. For marquee clones, add `inert={true}` (explicit, not bare `inert`, for React prop consistency). Radix's `Avatar` and similar preserve focusability.

## Heading order

- **The Saturn carousel needs an `sr-only <h2>`**. The section contains `<h3>` repo cards but sits between the hero `<h1>` and later `<h2>` sections. Without the sr-only heading, axe-core flags a "heading-order" violation. Add it in both mobile and desktop branches.

## Radix Dialog side effects

- **Open Radix Dialogs set `aria-hidden` on sibling page content.** This hides every other heading from `getByRole` in Playwright. Either dismiss the modal before asserting on background content, or use `page.locator("h1", { hasText: "..." })` to bypass the a11y-tree query.
- **Radix `DismissableLayer` swallows pointer events on any interactive element rendered *outside* the open dialog's portal tree.** You'll see the button render, hover-state highlights work, but `onClick` never fires. Fix: render interactive UI *inside* `DialogContent`. If the dialog is supposed to be "transparent" (e.g. takeover mode), set `className="pointer-events-none max-w-none w-auto bg-transparent border-0 p-0 shadow-none z-[60]"` on `DialogContent` and add `pointer-events-auto` only on the child cluster that must receive clicks. Keyboard shortcuts outside the dialog can be kept working by registering window listeners in the **capture** phase (they fire before Radix's own handlers can `preventDefault`).

## ARIA role promises

- **Don't slap `role="menu"` / `role="menuitem"` on a Popover unless you also wire arrow-key navigation, typeahead, and Escape between items.** Screen readers announce "menu" and users expect WAI-ARIA menu keyboard semantics; without them, the UI misleads. If the wrapper only needs to pair two actions, use `role="group"` with an `aria-label` — honest semantics that don't over-promise. Swap in `DropdownMenu` (Radix) when you want the real menu behavior.

## Plugin config

- **`eslint-plugin-jsx-a11y` flat config**: import as `jsxA11y` and add `jsxA11y.flatConfigs.recommended` to the extends array. Runs cleanly because shadcn files are already in `globalIgnores`.
